0427일의 작업 기록은 모델의 **'안정성 증명'**과 **'새로운 구조적 시도'**로 요약됩니다. 단순한 하이퍼파라미터 수정을 넘어, AutoML의 도입과 데이터 누수를 스스로 필터링하는 분석의 정교함을 더했습니다.

## 🚀 0427 사고의 흐름 (Evolutionary Logic)

### 📈 Phase 1: 안정성의 고도화 (V1 ~ V3)
- **V1 (Base Integration)**: 8조의 V9 피처 전략을 우리 파이프라인에 완벽히 이식하고, 잔차(Residual) 보정 로직을 통해 기본기를 다졌습니다.
- **V2 (Weight Optimization)**: CatBoost(0.6), LGBM(0.2), XGB(0.2)의 최적 가중치를 산출하여 앙상블의 균형을 맞췄습니다.
- **V3 (Multi-Seed Averaging)**: 모델이 난수(Seed)에 따라 성능이 출렁이는 것을 방지하기 위해 CatBoost 5개 시드 앙상블을 적용, **'운'이 아닌 '실력'으로서의 AUC**를 확보했습니다.

### 🤖 Phase 2: 머신러닝의 한계 측정 (V4: AutoGluon)
- **접근**: 사람이 설계한 앙상블 외에, 최신 AutoML 기법인 **AutoGluon**이 찾는 최적의 스태킹 구조가 무엇인지 탐색했습니다.
- **성과**: 수십 개의 모델을 자동으로 스태킹하며 우리가 미처 발견하지 못한 피처 간의 고차원적 상호작용 가능성을 확인했습니다.

### 🧪 Phase 3: 미세 신호 포착과 실패의 기록 (V6 ~ V8)
- **V6/V7 (Clinical Tuning)**: 전문의 가이드라인을 더 공격적으로 반영했습니다. 특히 특정 폴드에서 0.7422를 상회하는 결과를 얻으며 가능성을 보았습니다.
- **V8 (The Critical Moment - Leakage Detection)**: 
    - AUC가 0.7740으로 급등하는 현상을 발견했습니다.
    - 기뻐하기보다 **'의학적으로 불가능한 수치'**임을 직시하고 SHAP 리포트를 통해 분석을 수행했습니다.
    - 결과적으로 특정 피처에서 발생한 **데이터 누수(Leakage)**를 확인하고 이를 다시 격리하는 자정 과정을 거쳤습니다.

# 🏥 IVF Project: Advanced Stacking & Hybrid Systems (0427 Part 2)
> **목표: 하이브리드 잔차 스태킹(Hybrid Residual Stacking)을 통한 최적의 일반화 성능 확보**

0427일 후반부 작업은 모델의 개수를 늘리는 단순 앙상블에서 벗어나, **모델 간의 상호 보완성(Complementarity)**을 극대화하는 아키텍처 설계에 집중했습니다.

---

## 🏗️ 0427 후반부 사고의 흐름 (Advanced Logic)

### 🧩 Phase 4: 메타 스태킹의 정교화 (V9 ~ V11)
- **V9-V10 (Meta-Learner Stacking)**: 7가지 개별 모델 조합을 로지스틱 회귀(Logistic Regression)로 결합하여 최적의 비중을 수학적으로 산출했습니다. SHAP을 통해 모델이 의학적 인과관계를 유지하고 있는지 끊임없이 검증했습니다.
- **V11 (Baseline Strengthening)**: 8조의 전략을 흡수하여 기초 체력(Base Models)을 강화했습니다. 10-Fold Stratified CV를 통해 데이터의 모든 조각을 학습에 활용했습니다.

### 📉 Phase 5: 잔차 학습의 도입 (V12 ~ V14)
- **V12 (Hybrid Residual Stacking)**: 기본 모델이 맞히지 못한 '오답(Residuals)'만을 따로 배우는 Regressor를 추가했습니다. 모델의 시야를 '정답'에서 '실수'로 확장한 터닝 포인트입니다.
- **V13 (Regularized Ridge Stacking)**: 스태킹 시 발생할 수 있는 과적합을 방지하기 위해 **RidgeCV**를 도입, 규제(Regularization)를 통해 모델의 유연성을 확보했습니다.
- **V14 (Refined Hybrid)**: 가중치를 수동으로 미세 조정하며 임상적 통찰과 수치적 최적화 사이의 균형을 맞췄습니다. (AUC 0.7392~0.7393 구간 안착)

### 🧬 Phase 6: 시드 다양성의 완성 (V15: 3-Brothers Strategy)
- **접근**: CatBoost 모델 3개를 각각 다른 시드(Seed)와 성격(Standard, Deep, Robust)으로 학습시킨 후, 잔차 보정 시스템과 결합했습니다.
- **의미**: "한 명의 천재보다 세 명의 협진이 낫다"는 원칙을 구현했습니다. 개별 모델의 편향(Bias)을 서로 상쇄시켜 0.7422 돌파를 위한 가장 견고한 기반을 마련했습니다.

## 🛠 주요 기술 스택 및 방법론 (0427 Core)

1. **Seed Averaging**: CatBoost 5-Seed 학습을 통해 Public LB 점수의 안정성(Robustness) 극대화.
2. **AutoML Stacking**: AutoGluon을 활용한 앙상블 상한선 벤치마킹.
3. **Leakage Audit**: SHAP 리포트를 활용하여 비정상적으로 높은 가중치를 가진 변수를 색출, 모델의 '정직한 성능'을 검증.

## 💡 Lessons Learned: "성능보다 중요한 것은 신뢰다"
0427의 기록 중 가장 가치 있는 것은 **V8의 실패 기록**입니다. 0.77이라는 수치에 현혹되지 않고, 데이터 누수를 찾아내기 위해 SHAP 그래프를 뜯어본 과정은 본 모델이 실제 의료 현장에서 사용될 때 필요한 **'설명 가능성(XAI)'**과 **'신뢰성'**을 확보하는 훈련이 되었습니다.

우리는 이제 8조가 도달한 0.7422가 단순한 숫자가 아니라, 모든 누수를 완벽히 차단한 상태에서 도달할 수 있는 매우 정교한 고지임을 이해하게 되었습니다.

## 🛠 핵심 아키텍처 요약 (Technical Stack)

1. **Residual Learning**: `y - pred` 값을 학습하여 예측치의 미세한 오차를 보정.
2. **Meta Stacking**: Logistic Regression 및 RidgeCV를 활용한 다중 모델 결합.
3. **Seed Averaging**: CatBoost의 확률적 변동성을 제어하기 위한 Multi-seed 전략.
4. **Leakage-Free Pipeline**: 모든 파생 변수 생성과 인코딩을 Fold 내부에서 수행하여 데이터 누수를 완벽히 차단.
