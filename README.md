# 🏥 IVF Pregnancy Success Prediction: Clinical Insights & Ensemble Strategy
> **난임 환자 맞춤형 임신 성공 여부 예측 모델 개발 프로젝트**

본 프로젝트는 마리아병원과 LG Aimers가 주관한 해커톤 데이터를 바탕으로, 난임 시술(IVF/DI)의 성공 가능성을 예측하는 최적의 알고리즘을 탐구한 기록입니다. 단순한 데이터 분석을 넘어, **'배아의 생존력'**과 **'시술 타임라인의 효율'**이라는 의학적 메커니즘을 데이터로 정교하게 치환하는 데 집중했습니다.

---

## 🎯 Project Philosophy: "데이터는 숫자가 아니라 환자의 연대기다"
본 프로젝트의 핵심 가설은 **"난임 시술의 성공은 단일 변수가 아닌, 전 시술 과정의 유기적인 전환 효율(Attrition Rate)에 달려 있다"**는 것입니다.

1. **도메인 중심 피처 엔지니어링**: 난자 채취부터 수정, 배양, 이식에 이르는 '배아 파이프라인'을 수치화했습니다.
2. **엄격한 데이터 방역**: 데이터 누수(Data Leakage)를 원천 차단하기 위해 모든 검증은 OOF(Out-of-Fold) 방식으로 진행했습니다.
3. **모델의 다양성(Diversity)**: CatBoost, XGBoost, LightGBM의 서로 다른 의사결정 특성을 결합하여 예측의 안정성을 확보했습니다.

## 🔍 기술적 개선 및 보완 사항 (Critical Analysis)

### 🛠 현재 적용된 강점
- **OOF Stacking Strategy**: 교차 검증 과정에서 생성된 예측치를 메타 러너로 결합하여 과적합을 방지함.
- **Domain-Specific Imputation**: 구조적 결측치(NaN)를 '정보 없음'이 아닌 '절차 생략'의 의미로 해석하여 정보화함.

### 🚀 추가 보완이 필요한 지점 (Future Works)
1. **Target Encoding의 정교화**: 현재의 Label Encoding을 넘어, K-Fold 내에서 누수를 방지한 `Smoothed Target Encoding`을 적용하여 범주형 변수의 변별력을 극대화할 필요가 있음.
2. **Pseudo-labeling**: 모델이 매우 높은 확신을 갖는 Test 데이터를 다시 학습에 활용하여 0.7422 이상의 고지를 점령하는 전략이 유효할 것으로 판단됨.
3. **시술 유형별 분리 모델**: '동결 주기'와 '신선 주기'의 메커니즘이 근본적으로 다르므로, 이를 별도의 파이프라인으로 학습시키는 전략 고려.
